# Vista de Ejecución

El siguiente escenario describe el proceso de registrar un nuevo producto en el sistema ERP.

1. El Administrador de Compras ingresa los datos del producto en la aplicación web.
2. La aplica<img width="765" height="398" alt="SECUENCIA" src="https://github.com/user-attachments/assets/28ebfc80-8497-4e54-9d87-112fd6fc27ba" />
ción web envía la información a la API del sistema.
3. La API valida los datos recibidos.
4. Si los datos son correctos, la API guarda el producto en la base de datos.
5. La base de datos confirma la creación del producto.
6. El sistema muestra un mensaje de éxito al usuario.

Este flujo permite asegurar que la información registrada sea válida y almacenada correctamente.
